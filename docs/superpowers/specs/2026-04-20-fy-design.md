# Fy — Backend & Infrastructure Design Spec
Date: 2026-04-20

## Overview

Fy is a multilingual AI agent (Malagasy / French / English) built for Malagasy people, powered by Google Vertex AI and hosted on Firebase. This spec covers backend and infrastructure only; UI is handled separately.

---

## Architecture

```
Client (Firebase Hosting)
        │
        │ HTTPS calls with Firebase ID token
        ▼
Firebase Cloud Functions (modular, one per capability)
  ├── login
  ├── chat
  ├── generateImage
  ├── generateVoice
  ├── processDocument
  ├── setLanguage
  └── cleanup (scheduled weekly)

Shared Library (lib/)
  ├── auth.ts          — AuthProvider interface + NameAuthProvider
  ├── firestore.ts     — typed Firestore helpers
  ├── vertexai.ts      — Gemini + Imagen clients
  ├── embedder.ts      — chunking, embedding, RAG retrieval
  ├── extractor.ts     — PDF/DOCX/TXT/image/audio extraction
  ├── tts.ts           — Google Cloud TTS wrapper
  ├── prompts.ts       — system prompts (mg/fr/en)
  └── middleware.ts    — shared auth verification

Firestore
  ├── users/{userId}
  ├── users/{userId}/messages/{id}
  └── users/{userId}/embeddings/{id}   ← vector field, native vector search

Firebase Storage
  ├── uploads/{userId}/{fileId}
  └── voices/{userId}/{messageId}.mp3

Vertex AI (key never leaves server)
  ├── gemini-2.5-flash-lite   — text, vision, document Q&A
  ├── text-embedding-004      — embeddings for RAG (768-dim)
  ├── imagen-3                — image generation
  └── Google Cloud TTS        — voice synthesis
```

## Auth Layer

Abstracted behind an `AuthProvider` interface for easy future migration to Google Auth or email+password.

```typescript
interface AuthProvider {
  login(credentials: unknown): Promise<string>  // returns Firebase custom token
  resolveUserId(credentials: unknown): string
}
```

**NameAuthProvider (current):**
- `userId = sha256(name.toLowerCase().trim())`
- Issues Firebase custom token via Admin SDK
- Creates/updates `users/{userId}` on first login

**Firestore user doc:**
```
users/{userId}: {
  displayName: string,
  language: "mg" | "fr" | "en",   // default: "mg"
  createdAt: timestamp,
  lastSeenAt: timestamp
}
```

All Cloud Functions verify the Firebase ID token before any operation. Firestore security rules enforce per-user data isolation as a second layer.

## RAG Memory System

### Write path (after every message + document upload)
1. Split text into chunks (~512 tokens, 50-token overlap)
2. Embed each chunk via `text-embedding-004`
3. Store in `users/{userId}/embeddings/{chunkId}`

```
embeddings/{chunkId}: {
  text: string,
  embedding: vector(768),
  sourceType: "message" | "document",
  sourceId: string,
  hitCount: number,      // incremented on retrieval, used for cleanup
  createdAt: timestamp
}
```

### Read path (on every chat call)
1. Embed incoming user message
2. Firestore `findNearest()` → top-8 chunks by cosine distance
3. Increment `hitCount` on retrieved chunks
4. Build prompt: `[MEMORY block] + [system prompt] + [last 10 messages] + [user message]`
5. Call Gemini, return complete response

Embeddings are language-agnostic (multilingual model), so RAG works across all three languages.

## Cloud Functions

### `login`
- Input: `{ name: string, language?: "mg"|"fr"|"en" }`
- Derives userId, creates/updates user doc, issues custom token
- Returns: `{ token: string, userId: string }`

### `chat`
- Input: `{ message: string, attachments?: [{ storagePath: string, mimeType: string }] }`
- Auth: verifies Firebase ID token
- Inline attachments (images/audio) passed directly to Gemini multimodal — no separate processing
- Returns: `{ text: string }`

### `generateImage`
- Input: `{ prompt: string }`
- Auth: verifies Firebase ID token
- Calls Imagen 3 → saves to Storage
- Returns: `{ imageUrl: string }`

### `generateVoice`
- Input: `{ messageId: string }`
- Auth: verifies Firebase ID token
- Fetches message text from Firestore → Google Cloud TTS → saves `.mp3` to Storage
- Returns: `{ audioUrl: string }` (signed URL, 1hr expiry)
- Only called on explicit user request

### `processDocument`
- Input: `{ storagePath: string, mimeType: string }`
- Auth: verifies Firebase ID token
- Extracts text by type → chunks → embeds → writes to Firestore embeddings
- Returns: `{ chunksCreated: number }`

| File Type | Extraction Method |
|-----------|------------------|
| PDF | `pdf-parse` |
| DOCX | `mammoth` |
| TXT | Direct read |
| Images | Gemini vision → description |
| Audio | Google Cloud Speech-to-Text → transcript |

### `setLanguage`
- Input: `{ language: "mg"|"fr"|"en" }`
- Updates `users/{userId}.language`
- Returns: `{ ok: true }`

### `cleanup` (scheduled weekly)
- Deletes embeddings older than 90 days with `hitCount === 0`
- Deletes Storage files older than 30 days with no associated active embeddings

## Security

- API key stored in Firebase Functions environment config — never in client code or committed to git
- All Functions verify Firebase ID token before any operation
- Firestore rules: users can only read/write their own `users/{userId}` subtree
- Storage rules: users can only read/write their own `uploads/{userId}/` and `voices/{userId}/`
- Signed URLs for audio (1hr expiry) — no permanent public access

---

## Scalability Notes

- Auth abstraction: swap `NameAuthProvider` for `GoogleAuthProvider` or `EmailAuthProvider` with zero business logic changes
- Vertex AI client centralized in `lib/vertexai.ts` — model swaps happen in one place
- RAG scales with Firestore native vector search — no external vector DB to manage
- Each Cloud Function scales independently
